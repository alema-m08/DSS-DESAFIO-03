TaskOrganizer
Manual de Funcionamiento y Validaciones Clave

Sistemas Evaluados: MVC Nativo (PHP 8.2) & Laravel 12

1. Introducción al Funcionamiento

TaskOrganizer es un gestor de tareas colaborativo implementado bajo dos arquitecturas distintas pero compartiendo el mismo diseño visual premium Elysian Deep Forest & Gold. El flujo básico consiste en:

  • Autenticación de Usuarios: Registro de nuevas cuentas e inicio de sesión para delimitar el espacio de trabajo.
  • Gestión de Tareas (CRUD): Creación, lectura, edición y eliminación de tareas específicas para el usuario logueado.
  • Actualización Asíncrona (AJAX): Cambio inmediato del estado de las tareas (pendiente / completada) desde el Dashboard sin recargar la página.

2. Validaciones más Importantes (Seguridad y Datos)

Las validaciones se ejecutan tanto del lado del servidor (PHP/Laravel) para máxima seguridad, como del lado del cliente (HTML5) para una experiencia fluida. A continuación, se detallan las reglas críticas aplicadas:

A. Registro de Usuario

Campo Validación Crítica Objetivo / Seguridad Comportamiento si Falla
Nombre Requerido. Tipo Texto. Evitar registros vacíos. Muestra alerta "El nombre es requerido".
Email Requerido. Formato de correo válido. Único en base de datos. Evitar correos corruptos y cuentas duplicadas. Alerta: "Este correo electrónico ya está registrado".
Contraseña Mínimo de 6 caracteres. Coincidencia obligatoria con confirmación. Fuerza longitud mínima de seguridad. Evita errores de tipeo. Alerta: "Las contraseñas no coinciden" o longitud insuficiente.

B. Inicio de Sesión (Login)

  • Campos Requeridos: Ambos campos (email y contraseña) no pueden enviarse vacíos.
  • Existencia de Cuenta: Se verifica si el correo electrónico existe en la base de datos.
  • Criptografía: La contraseña ingresada se verifica mediante la función nativa password_verify() contra el hash seguro encriptado con Bcrypt.

C. Gestión de Tareas

Operación Validación Aplicada Tipo de Control
Creación Título obligatorio (no vacío). Máximo 255 caracteres. Servidor e HTML5 (required).
Edición Propiedad de la tarea. El ID de la tarea debe pertenecer al usuario autenticado. Control de Autorización (SQL query condicionado por user_id).
Cambio de Estado Solo admite valores pendiente o completada. Validación estricta de array en MVC Nativo y regla in:pendiente,completada en Laravel.

3. Medidas de Seguridad Clave

Protección Contra Inyección SQL

Tanto en MVC Nativo como en Laravel, se utilizan **Consultas Preparadas (Prepared Statements)** mediante PDO y Eloquent ORM. Las variables del usuario nunca se concatenan directamente en la consulta SQL, previniendo ataques maliciosos de inyección de código.

Protección Contra Falsificación de Peticiones en Sitios Cruzados (CSRF)

La versión Laravel utiliza tokens CSRF en todos los formularios y peticiones AJAX (incluido el toggle de estados). Cualquier petición POST/PUT/DELETE sin un token válido es automáticamente bloqueada por el Middleware de Laravel, impidiendo ataques de terceros.

Encriptación de Contraseñas

Las contraseñas se almacenan únicamente en su formato hash cifrado de una sola vía (utilizando Bcrypt). Incluso en caso de un acceso no autorizado a la base de datos, las claves originales de los usuarios permanecen seguras.

4. Mecanismo AJAX (Cambio de Estado Asíncrono)

Cuando un usuario marca el checkbox de una tarea en el Dashboard, ocurre el siguiente flujo técnico:

  1. Interceptación de Evento: JavaScript detecta el cambio en el checkbox y bloquea la recarga de página estándar.
  2. Petición HTTP: Se dispara un Fetch asíncrono con método POST (MVC Nativo) o PATCH (Laravel) hacia la ruta de actualización.
  3. Verificación de Sesión y Propiedad: El servidor comprueba que el usuario está logueado y que la tarea le pertenece.
  4. Persistencia: Se actualiza el campo estado en la base de datos de MySQL.
  5. Respuesta Dinámica: El servidor responde en formato JSON. El frontend actualiza los contadores de tareas pendientes/completadas en la parte superior y despliega una notificación flotante (Toast) de éxito en la esquina inferior derecha.
